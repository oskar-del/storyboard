The Big Problem

The context window was never the real problem.

Everyone's talking about context limits. Longer windows, bigger models. But the real problem is what happens after the window ends — you start from zero. Again.

Storyboard breaks that loop. When a session fills up, you don't lose context — you reload it. One click. Full brief. The next chat starts knowing everything the last one knew.

The Problem

Every session end is
a four-layer wipe.

It's not just that the AI forgets. It's that you lose four things at once — and none of them are being captured anywhere.

💬
The WHY disappears
The conversation that produced the decision — the back-and-forth, the corrections, the moment something clicked. Gone. You remember the outcome. Not the reasoning.
🔗
The HOW disappears
What you tried first. What failed. What was adjusted. The full trail of execution — session after session — lost between windows.
🫧
The AI's reasoning disappears
The AI's own deliberation — why it went that direction, what it weighed, what it almost said instead. That thinking is gone the moment the session ends.
✕
The roads not taken disappear
What was explicitly ruled out and why. Three weeks later you try it again — and waste the same time re-discovering the same dead end.
🧠
The structure lives only in your head
Which project holds what, what's in progress, what the pipeline looks like. There's no dashboard for any of this. You carry it alone, every day. That's exhausting — and one bad day away from gone.

How it works

From zero context to
full project memory

Three steps. Under 10 minutes to set up. No accounts, no cloud sync, no vendor lock-in.

1

Connect your projects

Run node mcp-server.js and open the dashboard. Point it at your working folders. For existing projects, import_project.py rebuilds your history retroactively from git.

2

Capture as you work

The MCP server captures screenshots automatically. Use the FAB button to add decisions and ideas mid-session. Everything timestamped, searchable, permanent.

3

Seed any new chat

Hit "Seed New Chat." A complete digest — north star, recent sessions, open decisions, key context — copies to clipboard. Paste into Claude, GPT, or any AI. Instant full context.

Day 5 — Design coherence & UX polish
The "old pill" problem. All idea blocks still used the bright yellow/cream aesthetic from day 1 — jarring in a dark dashboard. Rebuilt them from scratch: dark card, amber left stripe, icon band. Period tabs were disappearing when you clicked a project — fixed to persist and actually filter. Every surface now reads from the same visual system.
Idea blocks redesigned Period tabs persist on project view Right panel polish
Day 5 insight: Coherence is trust. If one corner of the UI still looks like an early prototype, visitors mentally discount the whole product. Every pixel is a signal.
6
Apr 19
Day 6 — Design system · Search ⌘K · Block detail redesign
Thought like a Google design systems engineer. Every block now has a semantic type color — purple for sessions, amber for ideas, green for decisions, blue for discussions, red for rejections. Before: a decision and a session in the same project looked identical (both used project color). Now type is immediately readable. Added command palette search (⌘K) with type filters, arrow navigation, term highlighting, and decision-text matching. Rewrote the block detail overlay: summary first, then decisions and ideas, then related blocks panel. Carry forward now works from any block, not just seeded ones.
Semantic type color system Search ⌘K with type filters + arrow nav Block detail — related blocks panel AI usage bar in nav
Day 6 insight: A design system isn't about colors. It's about reducing cognitive load — every consistent signal is one less decision the user has to make while navigating.
6
Apr 19 · afternoon
Day 6 — Today digest · Morning kickoff · Settings rebuilt
The Today tab got a full layout: header with session/decision/idea counts, staggered card drop-in animation (the Tetris feel), and a morning kickoff card that replaces the empty state. First thing you see when there's nothing captured yet is a "Good morning" greeting with an inline textarea. Type your first decision, press Enter — it drops in as the first block of the day's story. Settings panel went from a single API key field to a 4-tab panel: Appearance (theme + compact mode toggle), Data (live storage stats, export JSON, import with merge, clear local), AI (key + MCP status), About (sprint info + GitHub link).
Today digest — full layout Morning kickoff card Block drop-in animation Settings: 4 tabs + data management
Afternoon insight: The first block of the day sets the narrative. A tool that asks "what's your first decision today?" changes how you start work — not reactive to notifications, but intentional from the first moment.
6
Apr 19 · evening
Day 6 — Task windows · Instruction manual · Autonomous polish
The project header now shows active task windows — parallel workstreams within a project (dashboard, strategy, landing-page, mcp-server, etc.). Each task card shows block count, automation indicator (pulsing green dot), and filters the feed on click. 32 existing blocks tagged with task fields across 6 workstreams. An instruction manual (MANUAL.html) was written during an autonomous session — every feature, why it exists, how it connects. Oskar's rule: when a feature ships, the manual updates too.
Task windows strip task field on blocks Instruction manual Autonomous polish session
Evening insight: Parallel workstreams are the real unit of AI-assisted work — not individual prompts. When you can see all six task windows at once, you stop asking "what am I working on?" and start asking "which thread needs attention right now?"

Apr 20
Day 7 — Auto session startup · Live blocks (Tetris mode)
The most-requested feature: Storyboard now opens every session. Claude calls session_start before saying anything else — reads your last session, key decisions, and hottest project, then asks one question: "Continue from X, or start something new?" No clipboard, no manual /storyboard. It just happens. Today tab shows the brief as a dismissable morning card. Also shipped live block logging (Tetris mode) — decisions appear in Today within 10 seconds as you work, not just at end-of-session.
Auto session startup session_start MCP tool Live blocks (Tetris mode) Morning brief card
Day 7 insight: The first 30 seconds of a session determine its quality. If you start with full context, you make better decisions. If you start cold, you spend the first 10 minutes reconstructing what you already know.
