Product 1
🔌
Storyboard Connect
One-click MCP connector UI
The Web 2.0 → AI bridge. Connect any tool to your AI environment without touching a config file.
The problem

Every professional has their work locked in Web 2.0 containers — Dropbox, Google Drive, Notion, Slack, Evernote. AI agents can access all of these via MCP, but setup requires editing JSON config files and restarting apps. An architect firm can't do that. Neither can an estate agent, a teacher, or a founder without a technical co-founder.

The gap nobody has filled
The MCP ecosystem has connectors for every major tool. The GUI to set them up in one click doesn't exist. Storyboard Connect is that GUI.
How it works
  • Click "Connect Dropbox" → OAuth in browser → agent maps folders to projects
  • "I found 8 project folders. Create a Storyboard project for each?" → Yes → done
  • Files, docs, and notes become context — referenced automatically in AI sessions
  • Ongoing sync: new files tagged to the right project without manual work
  • Connector marketplace: Dropbox, Drive, Notion, Slack, GitHub, Gmail, Evernote
Why this makes you sticky
Once your Dropbox is wired in, leaving Storyboard means losing every connection. Each connector added increases switching cost exponentially.
Product 2
🔭
Storyboard Lens
Multi-model intelligence layer
Run any task across all AI models simultaneously. Build a personal map of which model wins at which type of work — passively, from your own usage.
The problem

Every AI model has different strengths. Claude is better at reasoning and long-form content. GPT-4o wins on code. Gemini is faster for quick lookups. But nobody knows this from their own work — they just pick one model and stick with it. The data to make better decisions exists, but nobody is collecting it.

Why only Storyboard can do this
Storyboard already sees all your work across all sessions. No other tool has this cross-session, cross-model view. The intelligence builds passively — you don't have to do anything extra.
How it works
  • Run any saved prompt or skill → sends to Claude, GPT-4o, Gemini simultaneously
  • Side-by-side results — pick the best, Storyboard remembers the choice
  • After 10–20 comparisons: personal model map auto-generates
  • Re-run historical tasks when a new model launches — see if it improves your results
  • Storyboard holds the API keys — users don't need accounts on every platform

Product 3
📋
Storyboard Blueprints
Expert playbooks for any business type
The things experts know that beginners don't know to ask about. Every step, every connection, every "don't forget this" — already baked in.
The problem

AI is reactive — it answers what you ask. Someone building their first e-commerce store asks "how do I set up a website?" and gets a perfectly good answer. They never ask about the Facebook Pixel. They spend €2,000 on ads with no retargeting audience. An expert would have put the pixel in on day one before spending a cent. That gap — between what you know to ask and what you need to know — is where beginners fail even with AI.

The real moat
Blueprints are codified expert knowledge from real campaigns. Not theory — actual lessons from running 500+ real estate transactions, 13,000 lead gen sites, and agency clients across verticals. That expertise takes years to accumulate and can't be faked.
Example — "I want to sell T-shirts online"
  • Domain + Shopify setup (15 min)
  • Google Search Console + Analytics wired up before launch
  • Facebook Pixel installed before first ad spend ← most people miss this
  • Product photography spec sheet for Meta ads (aspect ratios, file sizes)
  • Email capture from day one — not after traction
  • Abandoned cart sequence (recovers ~15% of lost sales)
  • Retargeting audiences built while organic traffic grows
  • Canonical URLs set before Google indexes anything
Each blueprint maps to skills + connectors
Every step in a blueprint triggers the right Storyboard skill and, where available, the relevant connector — Shopify MCP, Meta Ads MCP, Search Console MCP. The playbook executes itself.
